## Backend Architecture

Clean architecture with layered separation:

```
app/
??? api/           # HTTP route handlers (thin controllers)
??? schemas/       # Pydantic request/response models
??? services/      # Business logic orchestration
??? models/        # Domain and ORM models
??? database/      # Connection and session management
??? auth/          # Authentication and authorization
??? monitoring/    # Metrics, health, observability
??? core/          # Config, logging, shared infrastructure
??? utils/         # Pure utility functions
```

### Request Flow

1. Request enters FastAPI middleware (CORS, logging)
2. Route handler validates input via Pydantic schemas
3. Service layer executes business logic
4. Response serialized and returned with appropriate status code

## Infrastructure

### Development

- Docker Compose orchestrates backend, frontend, and PostgreSQL
- Hot reload enabled for both services
- Environment variables loaded from `.env`

### CI/CD

- GitHub Actions runs on push/PR to `main` and `develop`
- Backend: lint, test, build verification
- Frontend: lint, production build

## Security Model (Planned)

- JWT bearer token authentication
- Role-based access control (admin, operator, viewer)
- API keys stored in environment variables, never committed
- CORS restricted to configured frontend origins

## Observability (Planned)

- Structured JSON logging via structlog
- Request ID propagation
- Health and readiness endpoints
- Model performance metrics dashboard
